Kimlik Doğrulama

Tüm API istekleri kimlik doğrulama gerektirir. API istekleri için iki yöntem desteklenmektedir:

1. API Anahtarı (API Key)

API anahtarını istek başlığında (header) X-API-KEY olarak gönderebilirsiniz.

X-API-KEY: your_api_key_here

2. Bearer Token

Bearer token'ı istek başlığında (header) Authorization olarak gönderebilirsiniz.

Authorization: Bearer your_token_here

Not: API anahtarınızı veya token'ınızı güvenli tutun ve paylaşmayın. Token'lar belirli bir süre sonra geçerliliğini yitirebilir.

Rate Limiting

API istekleri için rate limiting uygulanmaktadır. Dakikada maksimum 60 istek yapabilirsiniz.

Her API yanıtı, kalan istek sayısını X-RateLimit-Remaining başlığında içerir.

API Endpoint'leri

GET /api/news

Tüm haberleri listeler. Varsayılan olarak sayfalandırılmış ve en son yayınlanan haberler önce gelecek şekilde sıralanmıştır.

Query Parametreleri

Parametre Tip Gereklilik Açıklama
page integer Opsiyonel Sayfa numarası (varsayılan: 1)
per_page integer Opsiyonel Sayfa başına gösterilecek haber sayısı (varsayılan: 15, maksimum: 50)
sort string Opsiyonel Sıralama alanı (varsayılan: "published_at", seçenekler: "published_at", "views", "title")
direction string Opsiyonel Sıralama yönü (varsayılan: "desc", seçenekler: "asc", "desc")
status string Opsiyonel Yayın durumu (varsayılan: "published")

Hata Kodları

API, aşağıdaki HTTP durum kodlarını kullanarak hataları bildirir:

Durum Kodu Açıklama
400 Bad Request - İstek parametreleri hatalı veya eksik
401 Unauthorized - Kimlik doğrulama başarısız (geçersiz API anahtarı veya token)
403 Forbidden - İstenen kaynağa erişim yetkisi yok
404 Not Found - İstenen kaynak bulunamadı
422 Unprocessable Entity - Doğrulama hatası (giriş verileri geçersiz)
429 Too Many Requests - Rate limit aşıldı
500 Internal Server Error - Sunucu hatası

Hata Yanıtı Örneği

{
  "message": "Hata mesajı burada yer alacak",
  "errors": {
    "field_name": [
      "Bu alan için doğrulama hatası 1",
      "Bu alan için doğrulama hatası 2"
    ]
  }
}

Kullanım Örnekleri

cURL ile API Kullanımı

curl -X GET \
  'https://example.com/api/news?page=1&per_page=10' \
  -H 'X-API-KEY: your_api_key_here'

JavaScript (Fetch API) ile Kullanım

fetch('https://example.com/api/news', {
  method: 'GET',
  headers: {
    'X-API-KEY': 'your_api_key_here'
  }
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Hata:', error));

PHP ile Kullanım

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, 'https://example.com/api/news');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
curl_setopt($ch, CURLOPT_HTTPHEADER, array(
  'X-API-KEY: your_api_key_here'
));

$response = curl_exec($ch);
curl_close($ch);

$data = json_decode($response, true);
print_r($data);
